SmartPick Ecosystem

An Integrated Student-to-Manager Component Dispatch Architecture

📝 Project Overview & Intent

Traditional component inventory management in engineering laboratories frequently suffers from sorting bottlenecks, manual logbook discrepancies, and delayed retrieval times during practical exercises. SmartPick remedies this by integrating an intelligent inventory management interface with a centralized cloud database and hardware-automated storage indicators.

⚙️ The Core Operational Workflow

1
Digital Request (Student Role)

Students browse real-time hardware quantities, upload required circuit diagrams securely, and submit components requests for formal laboratory supervisor verification.

2
Verification & Audit (Supervisor Role)

Supervisors review incoming requests on a dedicated review panel, confirming compliance before changing status profiles to approved or providing targeted feedback notes.

3
Automated Retrieval (Manager Dispense Hub)

Upon approval, store managers execute the dispatch pipeline. Selecting Search triggers automated localization streams, while final token confirmation handles programmatic stock deduction.

🌐 System Architecture & Tech Stack

The technical blueprint leverages a modern three-tier application architecture to synchronize hardware states with cloud transitions instantly.

Frontend Layer

Vanilla HTML5, semantic CSS layout tokens, and modern JavaScript (ES Modules). Hosted globally on Vercel for continuous automated compilation pipelines.

Cloud & Database Engine

Powered by Supabase (PostgreSQL). Handles role-based authentication, object asset storage bucket channels, and live row mutation streams via WebSockets.

Hardware Integration

Physical ESP32 Microcontroller nodes driving addressable LED lighting arrays. The controller listens to real-time changes over the network to automate hardware drawer lookups.

🛠️ Advanced System Implementation Tokens

  • Live Safety Interlocks: A global database gate variable state checking routine (`is_open`) prevents unauthorized hardware signals from running when the lab is closed.
  • Transactional Stock Integrity: Automated inventory processing loops parse arrays to ensure quantities do not fall below zero, triggering auto-declines if allocations shift mid-session.
  • Role Selection Enforcement: Secure route tokens dynamically distribute matching layout access controls specifically for Student, Supervisor, or Manager profiles.
